Reports and LLM analysis

Reports should explain what happened, why it matters, and what to do next.

Deterministic findings first

Aegrail findings are produced by rules and correlation, not by the model. The model can help summarize and reason about context, but deterministic evidence remains the source of truth.

LLM analysis

LLM prompts are selected by issue type and application profile. A PrestaShop module change, WordPress admin login, Mautic user change, Laravel auth event, and Yii2 RBAC assignment should not get the same generic prompt. The model output is formatted as readable HTML and escaped before rendering.

Advisory by design: LLM output can be useful, but it should never silently close or suppress deterministic findings.

Report shape

  • Executive summary with plain-language risk.
  • Affected company, site, environment, app, service, and node.
  • Timeline of linked signals.
  • Evidence table with hashes, timestamps, paths, users, IPs, and redacted metadata.
  • Comments, actions taken, ignored paths, deployments, and remaining work.

Notifications

Email and browser notifications are useful for high-severity findings and operational changes. SMTP delivery can use any configured SMTP provider. Browser notifications work for dashboard users who opt in from a supported browser session.
