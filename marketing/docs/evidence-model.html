Security model

Evidence model

Aegrail sends enough context to investigate changes without shipping private site secrets to the Hub.

Signals and findings

A signal is a normalized observation: file changed, user count changed, browser script observed, admin login detected, config coverage checked. A finding is a grouped issue created from one or more related signals.

Agent to Hub wire protocol

Agent traffic uses the Aegrail wire protocol. The Hub issues node credentials, stores the node public key, and returns the node private key once during provisioning. Events are encoded as JSON and protected with X25519 key agreement, HKDF, AES-256-GCM, associated data, and timestamp replay checks.

agent evidence JSON
  -> redaction and normalization
  -> encrypted wire envelope
  -> Hub decrypts and validates node identity
  -> PostgreSQL stores signals, findings, inventory state

Redaction

Agents and Hub use matching redaction rules for API keys, authorization headers, cookies, passwords, tokens, sessions, DSNs, and common credential assignments. File events send metadata and hashes, not file contents. LLM evidence bundles are redacted before analysis.

Timestamps

Aegrail tries to preserve when something happened, not only when it was discovered. File events include filesystem timestamps and change metadata. Database profiles use application timestamps when available and still compare against baseline state to catch suspicious changes.

Local queue

Agents queue events locally before sending. Queue files are written durably, sent events can be retained for a configured time, and failed sends remain pending with Hub error details so an operator can see what is blocked.
