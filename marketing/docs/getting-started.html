Getting started

Set up the Hub, create inventory, deploy one agent, and turn the first scan into a clean baseline.

The shape of Aegrail

Aegrail has two main parts: Hub and Agent. The Hub stores inventory, findings, signals, reports, users, 2FA, and notification settings. Agents run near the websites and send encrypted evidence to the Hub.

First steps

  1. Create the first owner user in the Hub. First login starts without 2FA, then asks the owner to enroll TOTP.
  2. Create a company, site, environment, app, service, and node in the dashboard.
  3. Copy the generated agent config. Make sure the site app value matches the Hub app slug exactly.
  4. Set real paths, URLs, database DSNs through environment variables, and log paths.
  5. Run the agent once, then continuously after the config is clean.
  6. Mark the first known-good scan as the initial safe state.

Initial safe state

The first scan may find old files, old users, noisy logs, and historical platform state. Review it, then mark the current state as safe. After that, Aegrail should focus on what changes next.

Rule of thumb: initial state is for known existing reality; findings after that are the things you need to explain.

What to check next

Node status

Confirm files, DB, logs, browser, and config collectors are reporting or intentionally disabled.

Findings

Open issue detail pages, inspect evidence, and use ignore or deployment marking for expected changes.
