Command reference

The codebase is split into Hub and Agent binaries, so operations stay explicit.

Hub

aegrail-hub serve
aegrail-hub db migrate
aegrail-hub version

serve starts the API and dashboard routes. db migrate applies PostgreSQL migrations. Production should run migrations as a separate container or job before starting the API.

Agent

aegrail-agent run --config /etc/aegrail/agent.yaml
aegrail-agent run --config /etc/aegrail/agent.yaml --once
aegrail-agent validate --config /etc/aegrail/agent.yaml

run starts continuous collection. --once is useful for testing a config before leaving the agent alive. Validation should catch missing roots, unknown app kinds, bad schedules, and missing required identity fields.

Docker use

For minimal containers, prefer running commands through Compose instead of expecting shell tools inside the image.

docker compose --profile tools run --rm aegrail-hub-migrate
docker compose logs -f aegrail-hub
docker compose logs -f aegrail-agent-demo-site

Debug checklist

  • 404 from ingest usually means the agent identity does not match Hub inventory.
  • Browser rendered crawl failures usually mean Chrome is unavailable or not configured for that container.
  • Pending queue files mean the agent collected data but the Hub did not accept or receive it yet.
