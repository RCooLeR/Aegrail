01

Watch real changes

Track important files, plugins, modules, admin users, login events, password resets, scripts, configs, and access-log signals without turning normal cache churn into noise.

02

Keep private data private

Agents send hashes, timestamps, paths, normalized metadata, and redacted evidence. Passwords, DSNs, API keys, cookies, and tokens are scrubbed before they leave the node.

03

Decide quickly

The Hub turns raw signals into findings you can mark safe, ignore by rule or path, group into deployments, send to LLM review, and export into readable reports.

How it works

Simple monitoring, useful evidence

Aegrail is not trying to be a giant SIEM. It is a practical way to see whether the client sites you care about are still in the state you expect.

Step 1

Create inventory

Create a company, site, environment, app, service, and node in the dashboard.

Step 2

Install agent

Copy the generated node config, adjust paths and URLs, and run the agent near the site.

Step 3

Mark initial state

Use the first clean scan as the baseline so old noise does not keep shouting at you.

Step 4

Review findings

Open the issue detail page, inspect evidence, run LLM analysis, comment, ignore, or report.

Supported apps

Focused on the stacks we actually monitor.

Each application profile keeps the collector high-signal: WordPress, PrestaShop, Mautic, Laravel, Yii2 RBAC, static, React, and Node.js all have different paths, tables, noise patterns, and useful security markers.

WordPressPrestaShopMauticLaravelYii2 RBAC
What it watches
FilesHashes, ctime, mtime, size, module/plugin changes, suspicious PHP guards.
DatabaseUsers, roles, important options, modules/plugins, app-specific auth tables.
LogsAdmin logins, reset requests, suspicious requests, Tor IP checks, noisy static paths filtered.
BrowserClient-visible scripts, favicons, links, headers, and optional rendered crawls when Chrome is available.
